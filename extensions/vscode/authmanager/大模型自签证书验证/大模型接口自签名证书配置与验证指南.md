# 大模型接口自签名证书配置与验证指南

在私有化部署或使用自签名证书的大模型接口时，Continue 插件默认会因为安全校验而拒绝连接。本文档介绍如何配置跳过校验以及如何验证配置是否生效。

## 一、 如何设置（禁用 SSL 校验）

你可以通过修改 Continue 的配置文件（`config.json` 或 `config.yaml`）来禁用 SSL 校验。

### 1. 全局设置（推荐）
在配置文件的顶层添加 `requestOptions`，这对所有模型生效。

**YAML 格式 (`config.yaml`):**
```yaml
# 全局请求设置
requestOptions:
  verifySsl: false # 忽略 HTTPS 证书校验
```

**JSON 格式 (`config.json`):**
```json
{
  "requestOptions": {
    "verifySsl": false
  }
}
```

### 2. 局部设置（针对特定模型）
如果你只想针对某个模型跳过校验：

```yaml
models:
  - name: "My Private Model"
    provider: "openai"
    apiBase: "https://your-private-domain:8443/v1"
    requestOptions:
      verifySsl: false # 仅针对此模型关闭校验
```

---

## 二、 如何验证是否生效

为了确认配置已正确应用，可以通过开启详细日志模式进行验证。

### 1. 开启详细日志环境变量
在启动 VS Code 前设置环境变量 `VERBOSE_FETCH=true`。

- **Windows (PowerShell)**:
  ```powershell
  $env:VERBOSE_FETCH="true"
  code
  ```

### 2. 查看开发者工具日志
1. 在 VS Code 中按下 `Ctrl + Shift + P`。
2. 输入并选择 `Developer: Toggle Developer Tools`（开发者：切换开发人员工具）。
3. 切换到 **Console**（控制台）标签页。
4. 在控制台顶部的过滤器（Filter）框中输入 **`Fetch agent options`** 进行搜索。
5. 在 Continue 中选择对应的模型并发送一条聊天消息。

### 3. 核心验证指标
在搜索结果中展开日志内容，重点观察：

- **关键行**：`rejectUnauthorized/verifySsl: false`
- **含义**：如果显示为 `false`，说明插件已成功读取你的配置，并将在发起网络请求时跳过证书校验。
- **证书统计**：你还能看到 `Total CA certs` 等信息，确认自定义证书是否被正确加载。

---

## 三、 常见问题排查

- **404 错误**：如果日志显示 `404 Cannot POST /...`，请检查 `apiBase` 路径。`openai` provider 会自动在 `apiBase` 后添加 `/chat/completions`。
- **No chat model selected**：修改配置后，请在 Continue 侧边栏底部的下拉菜单中重新手动选择一次你的模型。
- **日志没输出**：确保已设置 `VERBOSE_FETCH=true` 并重启了整个 VS Code 进程（需完全退出后重新打开）。
